# AI_COMMUTE_CODE_RULE.md
> AI 출근 시뮬레이터 코드 작업 규칙 v1.0

## 목적
이 문서는 Codex 및 개발자가 프로젝트를 수정할 때 반드시 따라야 하는 규칙을 정의한다.

---

# 1. 수정 원칙

- 요청받은 Screen만 수정한다.
- 요청하지 않은 Screen은 절대 수정하지 않는다.
- 디자인 변경은 AI_COMMUTE_UI_SYSTEM.md를 따른다.
- 화면 연결은 AI_COMMUTE_SCREEN_FLOW.md를 따른다.
- UI는 AI_COMMUTE_COMPONENT_LIBRARY.md의 컴포넌트를 우선 재사용한다.

---

# 2. 금지 사항

❌ '최적화'를 이유로 화면 흐름 변경 금지
❌ 사용하지 않는 것처럼 보여도 임의 삭제 금지
❌ CSS 통합/정리 명목으로 스타일 변경 금지
❌ 함수명 변경 금지(요청 없는 경우)
❌ 파일 이동 및 구조 변경 금지

---

# 3. 허용 사항

✅ 버그 수정
✅ 요청된 화면의 UI 개선
✅ 컴포넌트 재사용
✅ 기존 디자인 시스템 적용
✅ 필요한 최소 범위 리팩터링

---

# 4. 작업 절차

1. 관련 문서 읽기
2. 수정 범위 확인
3. 영향 범위 분석
4. 해당 Screen만 수정
5. 다른 Screen 영향 여부 점검
6. 변경 사항 보고

---

# 5. 변경 보고 형식

- 수정한 Screen
- 수정한 파일
- 변경 내용
- 영향받는 기능
- 테스트 방법

---

# 6. 기능 연결 규칙

Screen Flow를 임의로 변경하지 않는다.

예)
S005 → S006 → S007 → S008

연결을 수정할 경우 변경 이유를 반드시 기록한다.

---

# 7. CSS 규칙

- 기존 클래스 우선 사용
- 중복 스타일 생성 최소화
- 공통 스타일은 재사용
- 새로운 클래스는 명확한 이름 사용

---

# 8. JS 규칙

- gameState 구조 유지
- 기존 이벤트 흐름 유지
- 전역 변수 추가 최소화
- 하드코딩 최소화

---

# 9. 작업 시작 프롬프트

작업 전 반드시 아래 문서를 읽는다.

1. AI_COMMUTE_UI_SYSTEM.md
2. AI_COMMUTE_SCREEN_FLOW.md
3. AI_COMMUTE_COMPONENT_LIBRARY.md
4. PROJECT_GUIDE_AND_REVISION.md

이후 요청받은 Screen만 수정한다.

---

# 10. 프로젝트 철학

이 프로젝트는 'AI 출근 시뮬레이터'이다.

현재 '본앤하이리'는 첫 번째 시나리오 데이터이며,
향후 여러 출근 목적지를 지원할 수 있도록 확장성을 유지한다.
