# 본앤하이리 출근 마스터 시뮬레이터 프로젝트 종합 안내서 🌿

## 1. 프로젝트 개요

**본앤하이리 출근 마스터 시뮬레이터**는 특수학교 전공과 학생 및 발달장애 청년들이 직장 생활의 핵심인 **출근 전 시간 관리**와 **자립적인 일상 준비**를 가상으로 훈련할 수 있도록 돕는 웹 기반 시뮬레이션 게임입니다.

이 프로젝트는 단순한 텍스트 퀴즈나 정답 선택형 게임이 아니라, 학생이 다음 정보를 종합하여 판단하도록 설계되어야 합니다.

- 현재 시간
- 출근 시간
- 남은 시간
- 준비물 상태
- 알람 설정
- 버스 도착 시간
- 버스 이동 시간
- 하차 후 도보 시간
- 직무 미션
- 지각 위험도

핵심 목표는 학생이 자신의 선택으로 인해 시간이 흐르고, 버스를 놓치거나, 지각하거나, 준비물을 빠뜨리는 결과를 직접 경험하게 하는 것입니다.

즉, 이 시뮬레이터의 최종 목표는 **“정답 맞히기 게임”** 이 아니라 **“출근 시간관리 훈련용 시뮬레이션”** 입니다.

## 2. 교육적 핵심 의도

이 프로젝트는 학생이 다음 내용을 자연스럽게 경험하도록 설계합니다.

```text
내가 행동을 선택한다.
그 행동에는 시간이 걸린다.
시간이 지나면 버스가 가까워진다.
준비를 오래 하면 버스를 놓칠 수 있다.
버스를 놓치면 다음 버스를 기다려야 한다.
버스를 타고도 이동 시간이 필요하다.
늦을 것 같으면 연락하거나 전략을 바꿔야 한다.
전날 준비가 다음날 출근 성공에 영향을 준다.
출근은 단순 이동이 아니라 직무 수행을 위한 준비이다.
```

학생에게 모든 원리를 말로 설명하기보다, 시뮬레이션 속에서 반복적으로 경험하게 하는 것이 핵심입니다.

## 3. 대상자 맞춤형 수준별 학습 구조

학생의 인지 및 기능 수준에 맞추어 UI와 제어 로직을 3단계로 구성합니다.

### 가 수준: 높은 자율성

- 텍스트 위주의 설명 제공
- 필수 준비와 선택 준비 모두 제공
- 자유로운 선택 가능
- 행동의 결과를 스스로 감당
- 알람 설정, 버스 시간 확인, 날씨 확인, 준비물 선택, 연락 방법 확인 등 다양한 선택지를 제공
- 시간관리와 우선순위 판단을 중심으로 설계

예시 탭:

```text
알람 맞추기
가방 챙기기
작업복 챙기기
버스 확인하기
날씨 확인하기
우산 챙기기
물통 챙기기
연락 방법 확인하기
명찰 확인하기
```

### 나 수준: 중간 지원

- 텍스트와 이모지 아이콘 병행 제공
- 필수 준비 탭 중심
- 필수 행동 누락 시 경고 제공
- 선택 기회를 한 번 더 제공
- 정해진 순서가 아니라, 학생이 준비 순서를 일부 선택할 수 있도록 구성

예시 탭:

```text
알람 맞추기
가방 챙기기
작업복 챙기기
버스 확인하기
```

### 다 수준: 강한 지원

- 직관적인 아이콘과 음성 안내 중심
- 큰 버튼 제공
- 순서형 시뮬레이션 유지
- 필수 행동을 완료하지 않으면 다음 단계로 넘어갈 수 없도록 안전망 제공
- 반복 훈련과 성공 경험 중심

예시 흐름:

```text
알람 맞추기
→ 가방 챙기기
→ 버스 확인하기
→ 다음 날 출근하기
```

## 5. 전날 저녁 준비 단계

성공적인 출근은 전날부터 시작된다는 것을 학생이 경험하도록 합니다.

전날 준비 단계에서는 다음 활동을 포함합니다.

### 5-1. 알람 설정

학생이 내일 출근 시간에 맞추어 알람을 설정합니다.

- 가 수준: 수동 조작 휠 또는 직접 선택
- 나/다 수준: 프리셋 버튼 제공

알람을 늦게 설정하면 다음 날 아침 시작 시간이 늦어지고, 지각 위험이 증가해야 합니다.

### 5-2. 미리 가방 싸기

학생이 내일 필요한 준비물을 미리 챙길 수 있도록 합니다.

예시 준비물:

```text
작업복
가방
명찰
교통카드
우산
물통
```

전날 가방을 미리 챙기면 다음 날 아침 준비 시간이 줄어듭니다.

### 5-3. 버스 시간표 확인

학생이 다음 날 타야 할 버스 시간을 미리 확인합니다.

버스 시간표를 확인하면 다음 날 아침에 버스 정보를 더 명확하게 볼 수 있습니다.

버스 확인을 하지 않은 경우에는 다음 날 버스 판단 정보가 제한되거나, AI 코치가 “버스 시간을 미리 확인하지 않았어요”라고 안내할 수 있습니다.

### 5-4. 날씨 확인

날씨 확인은 선택 준비로 넣을 수 있습니다.

날씨 확인 후 우산이나 물통을 준비하면 다음 날 상황에서 자동 해결 또는 긍정적 결과로 이어질 수 있습니다.

예시:

```text
비 예보 확인 + 우산 챙김
→ 다음 날 비 상황 자동 해결

더운 날씨 확인 + 물통 챙김
→ 이동 중 갈증 문제 예방
```

## 6. 아침 자유 루틴 화면

아침 준비 화면은 학생이 시간 압박을 가장 많이 느끼는 핵심 화면입니다.

이 화면에서는 현재 시간이 가장 중요하게 보여야 합니다.

### 6-1. 필수 표시 정보

아침 준비 화면에는 다음 정보를 크게 보여줍니다.

```text
현재 시간
출근 시간
남은 시간
다음 버스 시각
버스 도착까지 남은 시간
```

예시:

```text
현재 시간: 08:00
출근 시간: 09:00
남은 시간: 60분
다음 버스: 08:30
버스 도착까지: 30분
```

### 6-2. 선택 버튼 방식

버튼을 누르기 전에 최종 결과를 미리 알려주지는 않습니다.

학생이 실패를 경험하는 것도 중요한 학습이기 때문입니다.

다만 판단에 필요한 정보는 충분히 제공합니다.

선택 버튼 예시:

```text
🍳 아침밥 먹기 +20분
🚿 샤워하기 +15분
👕 외출복 입기 +5분
🎒 가방 확인하기 +2분
📱 스마트폰 확인하기 +10분
🚶 지금 출발하기
```

### 6-3. 행동 선택 후 피드백

학생이 행동을 선택하면 다음 요소가 함께 변해야 합니다.

- 행동 애니메이션 재생
- 현재 시간 증가
- 버스 미니맵 위치 변화
- 버스 도착까지 남은 시간 감소
- 버스를 놓쳤는지 여부 표시
- 다음 버스 정보 갱신
- 지각 위험 상태 갱신

예시:

```text
🍳 아침밥을 먹었습니다.

20분이 지났습니다.
현재 시간은 08:20입니다.

버스가 정류장에 가까워졌습니다.
다음 버스 08:30까지 10분 남았습니다.
```

버스를 놓친 경우:

```text
🚌💨 버스가 지나갔어요!

08:30 버스를 놓쳤습니다.
다음 버스는 08:45입니다.
```

## 7. 행동별 컷씬 애니메이션

학생이 행동을 선택하면 텍스트 결과만 보여주는 것이 아니라, 짧은 행동 애니메이션을 보여줍니다.

고급 3D 애니메이션은 필요 없으며, 로컬 환경에서 HTML, CSS, JavaScript만으로 구현 가능한 수준이면 충분합니다.

### 7-1. 우선 적용할 행동

```text
아침밥 먹기
샤워하기
옷 입기
가방 확인하기
스마트폰 보기
바로 출발하기
```

### 7-2. 애니메이션 예시

아침밥 먹기:

```text
아침밥 먹기 선택
→ 식탁/밥/캐릭터 이모지 또는 이미지 등장
→ 2~3초 동안 먹는 느낌의 애니메이션
→ 시계가 08:00에서 08:20으로 변화
→ 버스 미니맵에서 버스가 정류장 쪽으로 이동
→ 결과 안내
```

스마트폰 보기:

```text
스마트폰 보기 선택
→ 휴대폰 화면에 집중하는 장면
→ 시간이 빠르게 흐르는 연출
→ 버스가 정류장에 가까워지거나 지나감
→ 지각 위험도 증가
```

### 7-3. 애니메이션의 교육적 목적

애니메이션은 단순 꾸미기가 아니라 다음 개념을 시각화하기 위한 장치입니다.

```text
행동을 하면 시간이 지나간다.
시간이 지나면 버스가 가까워진다.
준비를 오래 하면 버스를 놓칠 수 있다.
내 선택이 다음 상황에 영향을 준다.
```

## 8. 5개 정류장형 버스 미니맵

아침 준비 화면에는 5개 정류장형 버스 미니맵을 표시합니다.

이 미니맵은 전체 출근 경로가 아니라, **다음 버스가 우리 정류장에 가까워지는 모습**만 보여줍니다.

### 8-1. 미니맵 구조

예시:

```text
🚌 다음 버스 08:30 도착
도착까지 25분

정류장1 ━ 정류장2 ━ 정류장3 ━ 정류장4 ━ 우리 정류장 🚏
```

### 8-2. 남은 시간에 따른 버스 위치

버스 아이콘은 남은 시간에 따라 위치가 변합니다.

예시 기준:

```text
30분 전: 버스가 가장 먼 정류장 근처
25분 전: 두 번째 정류장 근처
15분 전: 중간 지점
5분 전: 우리 정류장 바로 전
0분: 우리 정류장 도착
0분 이후: 버스가 지나감, 다음 버스 정보로 갱신
```

### 8-3. 버스 놓침 연출

학생이 아직 집에 있는데 버스 도착 시간이 지나면 버스를 놓치는 연출을 보여줍니다.

예시:

```text
🚌💨 버스가 지나갔어요!

08:30 버스를 놓쳤습니다.
다음 버스는 08:45입니다.
```

이 연출은 학생이 “내가 준비하는 동안 시간이 지나가서 버스를 놓쳤다”는 것을 직관적으로 이해하도록 돕습니다.

### 8-4. 디자인 방향

미니맵은 너무 단순한 텍스트가 아니라, 시뮬레이션 게임처럼 보이도록 구성합니다.

필수 요소:

```text
버스 아이콘의 부드러운 이동
정류장 아이콘 5개
현재 버스 위치 강조
우리 정류장 강조
남은 시간에 따른 색상 변화
버스 놓침 시 흔들림 또는 지나감 애니메이션
```

색상 기준 예시:

```text
여유 있음: 초록
서둘러야 함: 노랑
매우 급함: 빨강
버스 놓침: 회색 또는 검정
```

### 8-5. 적용 범위

버스 미니맵은 아침 준비 화면에서만 보여줍니다.

학생이 “지금 출발하기”를 누르고 집을 나선 뒤에는 미니맵을 숨기고, 장면 중심 출근 시뮬레이션으로 전환합니다.

## 9. 출근 경로 시뮬레이션

집을 나선 후에는 실제 출근 과정을 단계별로 보여줍니다.

### 9-1. 필수 이동 시간

다음 시간이 반드시 반영되어야 합니다.

```text
집에서 정류장까지 걷기: 10분
버스 대기: 버스 도착 시각에 따라 결정
버스 탑승 이동: 30분
하차 후 본앤하이리까지 걷기: 10분
```

### 9-2. 성공 예시

```text
08:00 집 출발
08:10 정류장 도착
08:15 버스 탑승
08:45 하차
08:55 본앤하이리 도착

결과: 출근 성공
```

### 9-3. 지각 예시

```text
08:15 집 출발
08:25 정류장 도착
08:30 버스 탑승
09:00 하차
09:10 본앤하이리 도착

결과: 지각
```

### 9-4. 핵심

버스를 타면 곧바로 도착하면 안 됩니다.

학생은 다음 질문을 경험해야 합니다.

```text
몇 시에 집에서 나가야 할까?
지금 출발하면 몇 시 버스를 탈 수 있을까?
버스를 놓치면 얼마나 늦어질까?
버스를 타고 이동하는 시간까지 계산하면 몇 시에 도착할까?
```

## 10. 본앤하이리 직무 미션 연계

출근은 단순 이동이 아니라 직무 수행을 위한 준비입니다.

따라서 시뮬레이터는 학생에게 “왜 출근해야 하는가”를 보여줘야 합니다.

### 10-1. 오늘의 직무 미션 예시

```text
🌱 화분 물주기
🥬 채소 정리하기
🌻 꽃밭 가꾸기
☕ 카페 정리하기
🧹 작업장 정리하기
```

### 10-2. 적용 위치

```text
시작 화면

성공 엔딩 화면
지각/실패 엔딩 화면

```

### 10-3. 성공 엔딩 연결 예시


## 11. AI 출근 코치

AI는 정답을 알려주는 힌트 버튼이 아니라, 현재 상태를 해석해주는 코치처럼 작동해야 합니다.

### 11-1. 역할 원칙

AI는 학생 대신 결정하지 않습니다.

AI는 판단에 필요한 정보를 정리해줍니다.

### 11-2. 예시

현재 상태:

```text
현재 시간: 08:20
다음 버스: 08:30
정류장까지: 10분
출근 시간: 09:00
```

AI 코치:

```text
지금 출발하면 08:30 버스를 탈 수 있습니다.
하지만 더 준비하면 버스를 놓칠 수 있습니다.
꼭 필요한 준비가 끝났다면 바로 출발하는 것이 좋습니다.
```

### 11-3. 지각 위험 시 예시

```text
현재 시간상 09:00 전 도착이 어렵습니다.
지금은 추가 준비보다 출발 또는 연락이 더 중요합니다.
늦을 것 같으면 선생님이나 직장에 상황을 말해야 합니다.
```

## 12. 사업장 도착 및 사회성 시나리오

본앤하이리에 도착한 뒤에는 사장님 또는 동료를 만나는 짧은 사회성 장면을 제공합니다.

예시:

```text
사장님이 “좋은 아침이에요”라고 말했습니다.

선택:
1. 밝게 인사하기
2. 고개만 끄덕이기
3. 아무 말 없이 지나가기
```

이 장면은 대인서비스 교과와 연결할 수 있습니다.

다만 전체 프로젝트의 중심은 시간관리이므로, 사회성 장면이 지나치게 길어지지 않도록 합니다.

## 13. 엔딩 화면 개선

학생에게는 먼저 게임 엔딩처럼 보이는 화면을 제공합니다.

자세한 보고서는 그 다음에 분리해서 제공합니다.

### 13-1. 성공 엔딩 예시

```text
🎉 출근 성공!

08:55 본앤하이리에 도착했어요.
오늘의 직무를 시작할 수 있어요.

오늘의 직무:
🌱 화분 물주기
🥬 채소 정리하기
```

버튼:

```text
결과 보기
다시 도전
처음으로
```

### 13-2. 지각 엔딩 예시

```text
😰 지각했습니다

09:20 본앤하이리에 도착했어요.
출근 시간보다 20분 늦었어요.
동료들이 먼저 일을 시작했어요.

가장 큰 원인:
🍳 아침밥 먹기 +20분
📱 스마트폰 확인 +10분
🚌 버스 놓침
```

버튼:

```text
AI 코치 조언 보기
결과 보기
다시 도전
```

### 13-3. 조건별 엔딩 유형

결과는 단순 성공/실패만이 아니라, 상태에 따라 다양하게 구성합니다.

예시:

```text
성공
지각
준비물 미흡
위생 불량
배고픔
비 젖음
출근 실패
```

### 13-4. 엔딩의 목적

학생이 “게임이 끝났다”는 느낌을 받아야 합니다.

결과 화면은 단순 보고서가 아니라, 학생이 자신의 선택 결과를 체감하는 시뮬레이션 엔딩이어야 합니다.

## 14. 학생용 결과와 교사용 보고서 분리

### 14-1. 학생용 결과 화면


예시:


```text
인쇄 기능
```

학생에게 처음부터 긴 보고서를 보여주지 말고, 교사용 리포트와 분리합니다.

## 16. 기술 스택 및 구조

```text
Frontend: HTML5, CSS3, Vanilla JavaScript
```

```text
js/engine.js
```

## 17. 구현 우선순위

한 번에 모든 기능을 구현하기보다 단계적으로 진행합니다.

### 1순위: 시간과 버스 계산 로직

```text
현재 시간
출근 시간
다음 버스 시간
버스 도착까지 남은 시간
집에서 정류장까지 걷는 시간
버스 이동 시간
하차 후 도보 시간
예상 도착 시간
```

### 2순위: 아침 준비 화면 개선

```text
현재 시간 강조
다음 버스 정보 강조
버스 미니맵 표시
행동 선택 후 시간 변화
버스 놓침 여부 처리
```

### 3순위: 5개 정류장형 버스 미니맵

```text
버스 위치 이동
남은 시간에 따른 색상 변화
버스 놓침 연출
다음 버스 정보 갱신
```

### 4순위: 행동 애니메이션

```text
아침밥 먹기
샤워하기
옷 입기
가방 확인하기
스마트폰 보기
바로 출발하기
```

### 5순위: 출근 경로 상세화

```text
집 → 정류장 도보
버스 대기
버스 탑승 및 이동
하차 후 본앤하이리 도보
최종 도착 판정
```

### 6순위: 엔딩 및 결과 분리

```text
학생용 엔딩
학생용 결과 카드
교사용 보고서
출근 전략 카드
```

### 7순위: 수준별 전날 준비 자유도 확장

```text
다 수준: 순서형
나 수준: 필수 탭
가 수준: 필수 + 선택 + 점검 탭
```

### 8순위: AI 코치 고도화

```text
정답 제시가 아닌 상태 해석
현재 시간/버스/준비물 기반 조언
지각 위험 시 대처 제안
```

## 19. 최종 목표

이 프로젝트의 최종 목표는 예쁜 선택지 게임이 아닙니다.

학생이 **시간을 보고, 준비 상태를 확인하고, 버스 시간을 고려하여, 실제 출근에 가까운 판단을 연습하는 것**입니다.

학생이 시뮬레이션을 마친 뒤 다음을 느낄 수 있어야 합니다.

```text
아침에 한 행동 때문에 시간이 지나갔다.
시간이 지나니까 버스가 가까워졌다.
너무 오래 준비하니까 버스를 놓쳤다.
버스를 놓치니까 출근 시간이 늦어졌다.
전날 준비를 잘하면 아침이 쉬워진다.
늦을 것 같으면 연락해야 한다.
출근은 일을 하기 위한 준비이다.
```

이 목표가 코드, UI, 애니메이션, 결과 화면, 보고서에 모두 반영되어야 합니다.
