# 본앤하이리 출근 마스터 시뮬레이터 개선 계획서

> 이 문서는 **본앤하이리 출근 마스터 시뮬레이터**의 전체 설계 기준 문서입니다.  
> 코덱스는 이 문서를 참고하되, 사용자가 요청한 단계만 수정해야 합니다.  
> 요청하지 않은 단계의 화면, 로직, 디자인은 임의로 변경하지 않습니다.

---

## 0. 프로젝트 핵심 목표

본 프로젝트는 전공과 학생들이 **출근 전날부터 실제 출근까지의 과정을 시뮬레이션**하며 시간관리, 준비물 확인, 버스 이용, 도착 후 회고를 연습할 수 있도록 돕는 교육용 웹앱이다.

핵심 목표는 다음과 같다.

1. 학생이 출근 준비 과정을 직접 선택하고 경험한다.
2. 선택 결과에 따라 성공, 지각, 출근 실패를 경험한다.
3. 단순한 정답 제공이 아니라, 생활 속 판단과 결과를 연결한다.
4. 가/나/다 수준별로 정보량, 선택지, 안내 방식을 다르게 제공한다.
5. 교사는 결과 리포트를 통해 학생의 핵심 어려움과 다음 지도 포인트를 확인한다.

## 2. 수준별 기본 원칙

### 가수준

- 읽기와 판단이 가능하므로 가장 많은 정보를 제공한다.
- 여러 선택지 중 직접 판단하게 한다.
- 시간 계산, 정류장 방향 선택, 버스 번호 선택, 하차벨 타이밍 판단을 포함한다.
- 오답 시 바로 실패가 아니라 힌트 후 재도전한다.
- 최대 2번까지 재도전 후 AI가 정답을 안내한다.

### 나수준

- 핵심 정보는 제공하되 문장보다 키워드, 큰 글씨, 아이콘 중심으로 제시한다.
- 선택지는 줄이고 힌트를 함께 제공한다.
- 1번 틀리면 힌트를 강화하여 다시 시도하게 한다.
- 다시 어려우면 정답을 안내한다.

### 다수준

- 정보는 한 번에 많이 보여주지 않는다.
- 단어, 그림, 큰 버튼, AI 음성 안내 중심으로 제공한다.
- 화면에는 단어와 그림을 보여주고, AI는 자연스러운 짧은 문장으로 읽어준다.
- [다시 듣기] 버튼을 제공한다.
- 오답 선택지는 거의 만들지 않고, AI 안내에 따라 진행하게 한다.

## 1단계. 기본 흐름 안정화

### 목표

새 기능과 디자인을 적용하기 전에 전체 화면 이동과 상태 저장이 끊기지 않도록 안정화한다.

### 작업 내용

- 현재 앱의 전체 화면 흐름을 확인한다.
- 사용하지 않는 과거 화면이나 레거시 흐름이 실수로 호출되지 않도록 정리한다.
- 학생 선택, 수준 분기, 상태 저장, 결과 기록이 유지되는지 확인한다.
- 기존 기능을 깨뜨리지 않고 현재 흐름이 정상 작동하도록 한다.

### 완료 기준

- 메인화면에서 결과 화면까지 기본 진행이 끊기지 않는다.
- 학생 이름, 수준, 선택 기록이 유지된다.
- 결과와 교사용 리포트 연결이 깨지지 않는다.

### 코덱스 작업 프롬프트

```text
BON_HIGHLEE_SIMULATOR_PLAN.md 문서를 먼저 읽고 전체 방향을 파악해 주세요.

이번 작업은 1단계 “기본 흐름 안정화”만 진행합니다.
새로운 기능이나 디자인을 추가하지 말고, 현재 앱의 화면 이동과 상태 저장이 깨지지 않도록 정리해 주세요.

확인할 내용:
1. 메인화면 → 학생 선택 → 출근 정보 → 전날 준비 → 아침 준비 → 버스 이동 → 결과 → 리포트 흐름이 끊기지 않는지 확인
2. 학생 이름, 수준, 선택 기록, 시간, 준비물 상태가 유지되는지 확인
3. 사용하지 않는 레거시 화면이나 이전 흐름이 실수로 호출되지 않도록 정리
4. 기존 결과 저장과 교사용 리포트 연결이 깨지지 않도록 유지

주의사항:
- 디자인 변경 금지
- 새 기능 추가 금지
- 요청하지 않은 단계 수정 금지
- 작업 후 어떤 파일을 수정했는지 요약해 주세요.
```

## 2단계. 데이터 구조 정리

### 목표

새로 확정된 출근 정보, 버스 정보, 준비물, 수준별 분기, 결과 피드백을 안정적으로 처리할 수 있도록 데이터 구조를 정리한다.

### 포함해야 할 주요 데이터

#### 학생 데이터

- 학생 이름 5명
- 수준: 가 / 나 / 다
- 캐릭터: 남자 / 여자

#### 출근 정보

1. 출근 시간
2. 날씨
3. 집에서 챙길 것
4. 오늘 할 일
5. 도착하면 사용할 것
6. 주의사항

#### 버스 정보

- 정류장 이름
- 정류장 방향
- 정류장까지 걷는 시간
- 탈 버스 번호: 200번
- 헷갈리는 버스 번호: 999번, 119번
- 버스 도착 시간
- 버스 이동 시간
- 내릴 정류장
- 내린 뒤 걷는 시간

#### 준비물

기본 물건:

```text
교통카드
휴대폰
물병
```

상황별 물건:

```text
비: 우산
추움: 장갑, 목도리
더움: 손선풍기
미세먼지/감기: 마스크
```

가방에 넣지 않는 물건:

```text
명찰
모자
앞치마
작업장갑
작업복
농장도구
카페도구
```

도착하면 사용할 것:

```text
앞치마
작업장갑
작업도구
```

### 코덱스 작업 프롬프트

```text

정리할 데이터:
1. 학생 5명 정보: 이름, 수준, 남자/여자 캐릭터
2. 출근 정보: 출근 시간, 날씨, 집에서 챙길 것, 오늘 할 일, 도착하면 사용할 것, 주의사항
3. 버스 정보: 정류장, 정류장 방향, 버스 번호 200/999/119, 버스 시간, 내릴 정류장

   - 기본 물건: 교통카드, 휴대폰, 물병

- 기존 gameState와 충돌하지 않게 정리해 주세요.
- 작업 후 새로 만든 데이터 구조와 수정 파일을 설명해 주세요.
```

## 3단계. 버스 이동 파트 재구성

### 목표

기존 클릭 중심 버스 이동을 실제 버스 이용 훈련처럼 재구성한다.

### 최종 흐름

```text
집 출발
↓
정류장 방향 선택
↓
정류장까지 걷기
↓
버스 번호 선택
↓
200번 버스 타기
↓
현재 정류장 확인
↓
하차벨 누르기
↓
버스에서 내리기
↓
미니 지도에서 본앤하이리 찾기
↓
회사까지 걷기
↓
도착
```

### 수준별 흐름

#### 가수준

- 집에서 정류장까지 걷기부터 시작한다.
- 정류장 방향을 직접 선택한다.
- 버스 번호 200 / 999 / 119 중 선택한다.
- 버스 안에서는 현재 정류장 글씨가 실시간처럼 바뀐다.
- 본앤하이리 앞 정류장이 되었을 때 하차벨을 누른다.
- 내린 뒤 미니 지도에서 본앤하이리를 직접 선택한다.

#### 나수준

- 정류장 도착부터 시작한다.
- 버스 번호는 200번 중심으로 보여주고 헷갈리는 선택지는 1개 정도만 제공한다.
- 버스 안에서는 팝업으로 “현재 정류장은 ○○입니다. 내릴까요?”를 보여준다.
- 네 / 아니오로 선택한다.
- 내린 뒤 미니 지도와 간판 힌트를 보고 본앤하이리를 선택한다.

#### 다수준

- 정류장 도착 + AI 안내부터 시작한다.
- AI가 “200번 버스를 타요.”라고 안내한다.
- 큰 버튼으로 진행한다.
- 버스 안에서는 AI가 “곧 내려요. 벨을 눌러요.”라고 안내한다.
- 미니 지도 선택은 최소화하고 [걸어가기] 버튼 중심으로 진행한다.

### 오답 처리

- 가수준: 2번까지 재도전, 이후 AI가 정답 안내
- 나수준: 1번 틀리면 힌트 강화, 이후 정답 안내
- 다수준: 오답 선택지를 거의 만들지 않음

### 코덱스 작업 프롬프트

```text
BON_HIGHLEE_SIMULATOR_PLAN.md 문서를 참고해 주세요.

이번 작업은 3단계 “버스 이동 파트 재구성”만 진행합니다.
다른 화면은 가능한 건드리지 말고, 버스 이동 흐름만 수정해 주세요.

구현할 흐름:
집 출발 → 정류장 방향 선택 → 정류장까지 걷기 → 버스 번호 선택 → 200번 버스 타기 → 현재 정류장 확인 → 하차벨 누르기 → 버스에서 내리기 → 미니 지도에서 본앤하이리 찾기 → 회사까지 걷기 → 도착

수준별 요구:
- 가수준:
  1. 정류장 방향 선택 포함
  2. 200번 / 999번 / 119번 중 버스 번호 선택
  3. 현재 정류장 글씨가 실시간처럼 변경
  4. 본앤하이리 앞 정류장에서 하차벨 누르기
  5. 미니 지도에서 본앤하이리 직접 선택
- 나수준:
  1. 정류장 도착부터 시작
  2. 200번 중심 + 헷갈리는 선택지 1개
  3. 정류장마다 팝업으로 “현재 정류장은 ○○입니다. 내릴까요?” 표시
  4. 네 / 아니오 선택
  5. 미니 지도 + 간판 힌트
- 다수준:
  1. AI 안내 중심
  2. 큰 버튼 중심
  3. 오답 선택지는 거의 만들지 않음

오답 처리:
- 가수준은 2번까지 재도전 후 정답 안내
- 나수준은 1번 틀리면 힌트 강화 후 정답 안내
- 다수준은 안내형으로 진행

주의사항:
- 실시간 버스 API 연결 금지
- 샘플 데이터로 구현
- 버스 번호는 200번을 정답으로 사용
- 999번, 119번은 헷갈리는 버스로 사용
- 기존 결과 기록과 연결될 수 있도록 실수 기록을 남겨 주세요.
```

## 4단계. AI 출근 정보 + 버스 정보 확인 화면

### 목표

전날 출근 정보 확인 화면을 수준별로 정리하고, 버스 정보를 현실적으로 제공한다.

### 출근 정보 순서

```text
1. 출근 시간
2. 날씨
3. 집에서 챙길 것
4. 오늘 할 일
5. 도착하면 사용할 것
6. 주의사항
```

### 버스 정보

```text
1. 가야 할 정류장
2. 탈 버스 번호
3. 버스 도착 시간
4. 내릴 정류장
5. 이동 시간
```

### 다수준 정보 제공 방식

- 한 번에 보여주지 않는다.
- 정보를 하나씩 보여준다.
- AI가 자연스러운 문장으로 읽어준다.
- [다시 듣기] 버튼과 [확인] 버튼을 제공한다.

예:

```text
화면: 10시
AI 음성: 오늘의 출근 시간은 10시입니다.
[다시 듣기] [확인]
```

```text
화면: 200번
AI 음성: 200번 버스를 타요.
[다시 듣기] [확인]
```

```text
옷 입기
```

- 선택
- 짧은 컷신
- 완료 팝업
- 시간 증가
- 현재 시각 확인

- 나수준: 주요 활동은 거의 완료해야 출발 가능

- 나수준: 가방 물건 확인 + 힌트

## 7단계. 결과 화면 + 상황별 애니메이션

### 목표

결과 화면 전에 학생의 선택 결과를 짧은 장면으로 보여주고, 최종 결과와 아쉬운 점 1개를 제공한다.

### 결과 종류

```text
출근 성공
지각
출근 실패
```

### 최종 결과 구조

```text
도착 또는 실패 상황 발생
↓
상황별 짧은 애니메이션
↓
출근 결과
↓
가장 중요한 아쉬운 점 1개
↓
내일 다시 해볼 점 1개
```

### 중요 원칙

- 아쉬운 점은 1개만 보여준다.
- 여러 문제가 있어도 우선순위가 가장 높은 것 1개만 보여준다.
- 교통카드 없음은 결과 화면의 아쉬운 점이 아니라 과정 중 이벤트로 처리한다.
- 교통카드가 없으면 버스를 타기 전에 집으로 돌아가서 챙기고, 시간이 추가 소요된다.

### 상황별 애니메이션 예시

- 밥 안 먹음: 배고파하는 장면
- 샤워 안 함: 몸이 찝찝한 장면
- 양치 안 함: 입 냄새가 날 수 있는 장면
- 우산 없음: 비 맞고 불편한 장면
- 늦잠: 시계를 보고 놀라는 장면
- 스마트폰 오래 봄: 시간이 지나가는 장면
- 버스 번호 실수: 다시 확인하는 장면
- 하차벨 실수: 내릴 정류장을 놓칠 뻔한 장면

```text
- 출근 성공

   - 버스를 타기 전에 교통카드가 없으면 집으로 돌아가서 챙김
   - 시간 추가 소요
   - 최종 결과는 시간에 따라 성공/지각/실패로 결정
```

## 4. 작업 운영 원칙

코덱스에게 1단계부터 9단계까지 한 번에 시키지 않는다.

권장 운영 방식:

```text
1. 이 MD 파일을 코덱스에게 읽힘
2. 1단계 프롬프트만 실행
3. 완료 후 직접 테스트
4. 문제가 있으면 수정
5. 다음 단계 프롬프트 실행
6. 반복
```

각 단계 작업 시 반드시 다음 문장을 포함한다.

```text
이번 작업은 ○단계만 진행합니다.
다른 단계의 화면, 로직, 디자인은 임의로 수정하지 마세요.
```

## 5. 절대 지켜야 할 것

1. 전체 흐름을 임의로 바꾸지 않는다.
2. 요청하지 않은 단계는 수정하지 않는다.
3. 실시간 버스 API는 연결하지 않는다.
4. 현재는 샘플 데이터 기반으로 구현한다.
5. 다수준은 큰 버튼, 그림, AI 음성 중심으로 구성한다.
6. 교통카드 없음은 결과 피드백이 아니라 과정 중 이벤트로 처리한다.
7. 아쉬운 점은 결과 화면에서 1개만 보여준다.
8. 교사용 리포트는 작은 버튼을 눌렀을 때만 보인다.
9. 메인화면은 학생 5명 카드형 UI로 구성한다.
10. 전체 디자인은 기존 시안 분위기와 맞춘다.
