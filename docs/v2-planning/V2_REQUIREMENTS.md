# V2 Requirements

## 범위 태그

- `[core]`: v2 MVP에 반드시 구현한다.
- `[optional]`: v2 이후 확장 가능하며 핵심 제출 흐름에는 넣지 않는다.
- `[legacy/exclude]`: v2 핵심 구현 대상에서 제외한다.

## [core] 핵심 흐름 확정

v2 MVP의 핵심 제출 흐름은 아래 순서로 고정한다.

학생 선택 -> 브리핑 -> AI 계획 -> 전날 준비 -> 수면 -> 알람 -> 아침 준비 -> 가방 챙기기 -> 통근 -> 결과 -> 교사용 리포트

오늘의 직무 확인/직무 수행, 관리자/학생 관리, 레거시 시나리오 화면은 이 핵심 흐름에 포함하지 않는다. 해당 기능은 아래의 optional 또는 legacy 범위로 분리한다.

## 프로젝트 목적

현재 v1은 HTML/CSS/JavaScript 기반의 "AI 출근 시뮬레이터" 최종 제출본이다. v2는 동일한 기능, 화면 흐름, 게임 규칙, 데이터 구조를 React + Tailwind CSS 기반으로 재구성하기 위한 후속 버전이다.

v2의 목적은 새 기능 추가가 아니라 v1의 학습 경험을 빠짐없이 재현하는 것이다. 학생이 출근 정보를 확인하고, AI 출근 계획을 입력하고, 알람과 준비물을 챙기고, 아침 준비와 버스 이동을 거쳐 결과와 교사용 리포트를 확인하는 전체 흐름을 유지해야 한다.

## 핵심 게임 규칙

- 학생은 최대 5명까지 등록/관리된다.
- 학생 선택 후 캐릭터 성별/아바타를 확인하고 게임을 시작한다.
- 게임 시작 시 출근 시간, 날씨, 필수 준비물, 버스 정보가 결정된다.
- 오늘의 직무 2개는 v1 데이터로 보존하되 v2 MVP에서는 `[optional]` post-result 확장 데이터로 분리한다.
- 출근 목표 시간은 09:00, 10:00, 13:00 중 하나로 무작위 선택된다.
- 도착 판정 기준은 목표 출근 시간과 실제 도착 시간이다.
- 목표 시간까지 도착하면 성공이다.
- 목표 시간 이후 10분 이내 도착하면 지각이다.
- 목표 시간보다 10분을 초과하면 실패이다.
- v2는 저장/분석용 `rawJudgment`와 화면 표시용 `displayJudgment`를 분리한다.
- `rawJudgment = incomplete`인 경우 `displayJudgment = success`로 표시할 수 있다. 단, 필수 준비물 누락은 피드백/리포트 후보로 남는다.
- 행동은 시간 비용을 발생시키며 현재 시간이 분 단위로 증가한다.
- 선택 로그, 통근 선택 로그, 준비물 상태, 아침 활동 상태는 결과/리포트에 반영된다.

## 학생 수준별 차이

v1은 학생 수준을 세 단계로 다룬다.

- `가` 수준: 가장 많은 정보를 직접 판단한다. 출근 정보 입력, 알람 조정, 정류장 방향 선택, 하차 판단, 목적지 찾기에서 더 많은 선택권이 있다.
- `나` 수준: 일부 정보가 자동 채워지거나 선택지가 단순화된다. 버스 정류장/하차 판단은 안내와 질문 형태로 제공된다.
- `다` 수준: 텍스트와 단계가 더 짧고 시각/음성 안내가 강화된다. 버스 정보와 준비물은 단계별 확인, 읽어주기, 강조 표시 중심으로 제공된다.

유지해야 할 수준별 차이:

- 브리핑: `다` 수준은 한 번에 전체 정보를 보여주기보다 단계별 카드와 TTS 재생을 사용한다.
- AI 출근 계획: `가` 수준은 직접 입력 후 AI 계산 버튼과 완료 버튼을 거친다. `나/다` 수준은 기본값이 자동 입력되고 확인 중심이다.
- 전날 준비: `가` 수준은 직접 알람 시간을 10분 단위로 조정한다. `나/다` 수준은 추천 옵션을 선택한다.
- 가방 챙기기: `가` 수준은 필수/선택/방해물을 넓게 보여준다. `나` 수준은 필수와 일부 방해물이 보인다. `다` 수준은 필수 물건 중심으로 순서 안내 또는 예/아니오 질문이 제공된다.
- 통근: `가` 수준은 정류장 방향과 하차 타이밍을 직접 판단한다. `나` 수준은 현재 정류장에서 내릴지 질문한다. `다` 수준은 목표 행동이 더 직접적으로 안내된다.
- 결과: `다` 수준은 짧은 결과 문장과 TTS를 우선한다.

## 시간 진행 로직

기본 시간 값:

- 초기 현재 시간: 07:00
- 기본 목표 시간: 09:00
- 마감 시간: 목표 시간 + 10분
- 버스 배차 간격: 10분
- 집에서 정류장까지: 10분
- 버스 이동: 30분
- 정류장에서 회사까지: 10분
- 교통카드 미소지 시 집에 다녀오기: 20분
- 아침 전체 루틴 기준값: 47분
- 알람 추천 여유 시간: 목표 출근 시간 기준 약 80분 전
- AI 계획의 준비 시간: 40분
- AI 계획의 여유 시간: 10분

아침 준비 활동 시간:

- 샤워: 10분. 근거는 현재 v1 최종 플레이 흐름의 아침 준비 게임 실행값이다.
- 아침 먹기: 10분. 근거는 현재 v1 최종 플레이 흐름의 아침 준비 게임 실행값이다.
- 양치: 5분
- 옷 입기: 5분
- 가방 확인/챙기기: 5분
- 알람 설정 후 수면 전환 시 10분이 진행된다.
- 10분 더 자기 선택 시 알람 시간이 10분 늦춰지고, 한 번만 선택 가능하다.
- 10분 더 자기 선택 시 `wokeUpEarly = false`로 확정한다.
- 필수 준비물 완료 + 가방 완료 버튼 클릭 시 `bagChecked = true`로 확정한다.

통근 시간:

- 정류장까지 걷기 10분.
- 잘못된 버스를 보내고 기다릴 때 2분 증가.
- 실제 버스 탑승 시 다음 잡을 수 있는 버스 시간까지 대기 시간이 증가한다.
- 버스 노선 정류장은 탑승 후 10분, 20분, 30분 지점으로 진행된다.
- 하차 후 회사까지 10분 걷는다.

## 준비물 및 날씨 규칙

기본 집 준비물:

- 교통카드
- 스마트폰
- 물병

날씨별 추가 준비물:

- 비: 우산
- 더움: 휴대용 선풍기
- 추움/빙판/눈: 장갑, 겉옷
- 미세먼지/감기: 마스크
- 맑음: 추가 없음

직무별 workplace 준비물은 `[optional]` 직무/리포트 확장 데이터로 사용된다.

- 물 주기/정원/수확: 작업장갑, 농장 도구
- 동물 먹이 주기: 작업복, 작업장갑, 농장 도구
- 빵 포장/컵 정리: 앞치마, 카페 도구
- 청소/택배 상자 정리: 작업장갑, 작업 도구

가방 챙기기 방해물:

- 게임기
- 과자
- 장난감

## 버스 관련 규칙

고정 버스 정보:

- 올바른 정류장: 전주대학교 방향 정류장
- 잘못된 정류장: 반대 방향 정류장
- 올바른 버스 번호: 200번
- 방해 버스 번호: 999번, 119번
- 하차 목표 정류장: 본하이리
- 노선 정류장: 중앙시장(10분), 전주터미널(20분), 본하이리(30분)
- 목적지 선택지: 본하이리(정답), 동네 공원, 편의점

버스 진행 규칙:

- `가` 수준은 출발 정류장 방향을 직접 선택한다.
- 틀린 정류장을 반복 선택하면 AI 힌트가 올바른 방향을 제시한다.
- 버스 번호 선택 단계에서 방해 버스를 먼저 보여주고, 올바른 200번 버스를 고르게 한다.
- 교통카드가 없으면 집에 돌아가 20분을 소모하고 다시 버스를 탄다.
- 하차벨은 본하이리 정류장에서 눌러야 한다.
- 목적지 지도에서 본하이리를 찾아야 회사 도착 단계로 진행된다.

## 결과 판정 기준

최종 판정:

- 성공: 실제 도착 시간이 목표 출근 시간 이하.
- 지각: 실제 도착 시간이 목표 출근 시간보다 늦고 목표 + 10분 이하.
- 실패: 실제 도착 시간이 목표 + 10분 초과.
- 준비물 누락은 최종 판정에서는 성공으로 표시될 수 있지만, 피드백 후보와 교사용 리포트에는 기록된다.

피드백 우선순위:

1. 출근 이동 실패
2. 지각
3. 잘못된 버스 번호
4. 하차벨/하차 정류장 실수
5. 목적지 선택 실수
6. 늦잠
7. 스마트폰 지연
8. 날씨 준비물 누락
9. 아침 식사 누락
10. 샤워 누락
11. 양치 누락
12. 옷 선택 문제
13. 가방 준비물 누락

결과 화면은 결과 컷신, 별점, 목표/실제 도착 시간, 잘한 점, 아쉬운 점, 다음 개선 메시지, 선택 기록, 교사용 리포트 진입을 제공한다.

## 기능 범위

### [core] 반드시 유지해야 하는 기능

- 학생 선택 후 확인 모달과 캐릭터 선택.
- 학생 목록과 게임 히스토리 저장.
- 출근 경로 브리핑.
- AI 출근 계획 입력/자동 계산/결과 타임라인.
- 출근 정보 다시 보기 모달.
- 전날 준비: 알람 설정, 가방 챙기기, 버스 정보 확인.
- 수면 전환 애니메이션.
- 아침 알람: 바로 일어나기, 10분 더 자기.
- 아침 준비 미니게임: 샤워, 식사, 양치, 옷 입기, 가방 챙기기.
- 가방 챙기기에서 필수 물건, 상황 물건, 방해물 구분.
- 출근 이동: 정류장, 버스 번호, 교통카드 이벤트, 버스 내부, 하차벨, 목적지 지도.
- 결과 컷신과 결과 화면.
- TTS 읽어주기.
- AI 힌트/피드백 모달.
- 교사용 빠른 리포트 모달.
- 교사용 상세 리포트 화면: 개별 보고서, 전체 요약, 인쇄.
- 상단 HUD: 현재 시간, 남은 시간, 준비물 체크 상태.

### [optional] 확장 가능 기능

- 학생 관리: 추가, 수정, 삭제, 최대 5명 제한, 수준/성별/아바타 저장. 단, 핵심 플레이 흐름에서는 제외하고 teacher/settings로 분리한다.
- 오늘의 직무 확인/직무 소개.
- 결과 이후 직무 수행 체크리스트.
- 직무별 workplace 준비물 상세 사용.

### [legacy/exclude] v2 핵심 구현 제외

- 기존 레거시 시나리오 선택 화면.
- 기존 단계 전환 화면.
- 기존 단독 계획 화면.

## v2에서 개선 후보

아래 항목은 v1 코드를 변경하지 않고 v2 설계 시 검토할 후보이다.

- mojibake로 보이는 한글 텍스트 인코딩을 v2 데이터 파일에서 UTF-8로 정리.
- 상태 관리를 단일 React store로 분리하고 화면별 파생 데이터를 selector로 계산.
- 시간/버스/결과 판정 로직을 순수 함수로 분리해 테스트 가능하게 구성.
- 텍스트 데이터, 학생 수준별 문장, 이미지 경로, 준비물 규칙을 코드에서 분리.
- 아침 준비 시간 정의가 데이터와 실행 코드에서 다르게 보이는 부분을 v2 요구사항에서 하나로 확정.
- 접근성: 버튼 aria-label, 키보드 이동, 모달 포커스 트랩 강화.
- TTS 재생/중지 상태를 React 훅으로 정리.
- 이미지 로딩 실패 시 fallback을 일관화.
- 교사용 리포트 데이터를 HTML 문자열 생성 대신 구조화 데이터 + 컴포넌트 렌더링으로 전환.

## 자체 검증

- 화면 누락 여부: `[core]` 학생 선택, 브리핑, AI 계획, 전날 준비, 수면 전환, 아침 알람, 아침 준비, 가방 챙기기, 통근, 결과, 리포트를 모두 포함했다.
- 범위 분리 여부: 직무와 관리자 화면은 `[optional]`, 레거시 시나리오 화면은 `[legacy/exclude]`로 분리했다.
- 기능 누락 여부: TTS, AI 힌트, localStorage, 인쇄, 결과 저장, 교통카드 이벤트, 10분 더 자기, 수준별 분기를 포함했다.
- 로직 누락 여부: 목표/마감 판정, 버스 시간표, 준비물/날씨, 피드백 우선순위, `rawJudgment`/`displayJudgment` 결과 분리를 포함했다.
- 데이터 누락 여부: 준비물, 방해물, 버스 번호, 정류장, `[optional]` 직무, 결과 피드백 데이터가 다른 문서와 연결되도록 기록했다.
