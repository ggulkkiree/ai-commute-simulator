# Screen Flow

## 범위 태그

- `[core]`: v2 MVP에 반드시 구현한다.
- `[optional]`: 나중에 확장 가능하며 핵심 제출 흐름에는 넣지 않는다.
- `[legacy/exclude]`: v2 핵심 구현 대상에서 제외한다.

## [core] 핵심 흐름 확정

1. 학생 선택 화면
2. 학생 확인 모달
3. 출근 경로 브리핑
4. AI 출근 계획 입력
5. AI 출근 계획 결과
6. 전날 준비 화면
7. 수면 전환 화면
8. 아침 알람 화면
9. 아침 준비 화면
10. 가방 챙기기 미니게임
11. 출근 이동 화면
12. 결과 컷신
13. 결과 화면
14. 교사용 빠른 리포트 모달
15. 교사용 상세 리포트 화면

핵심 제출 흐름은 다음 문장으로 고정한다.

학생 선택 -> 브리핑 -> AI 계획 -> 전날 준비 -> 수면 -> 알람 -> 아침 준비 -> 가방 챙기기 -> 통근 -> 결과 -> 교사용 리포트

## [optional] 분리 흐름

### [optional] post-result 직무 흐름

- 오늘의 직무 확인/직무 소개
- 결과 이후 직무 수행 체크리스트
- 직무 완료 후 처음으로 이동

이 흐름은 v2 MVP 핵심 출근 시뮬레이션 판정에 포함하지 않는다.

### [optional] teacher/settings 흐름

- 관리자/학생 관리 화면
- 학생 추가/수정/삭제
- 수준/성별/아바타 관리

이 흐름은 교사용 설정/운영 기능이며 핵심 플레이 흐름에 포함하지 않는다.

## [legacy/exclude] 제외 흐름

v1에 남아 있는 이전 시나리오 화면(`screen-game`, `screen-phase-transition`, `screen-planning`)은 호환 코드가 존재하지만 현재 주요 제출 흐름이 아니다. v2 MVP 핵심 구현 대상에서 제외한다.

## 화면별 목적과 버튼

### 1. [core] 학생 선택

- 화면 ID: `screen-main`
- 목적: 등록된 학생 중 플레이할 학생 선택.
- 주요 데이터: 학생 ID, 이름, 수준, 성별/캐릭터, 아바타.
- 버튼:
  - 출근 시작하기
  - 교사용 설정
  - 내용 안내
- 이동:
  - 학생 카드 선택 -> 학생 확인 모달
  - 교사용 설정 -> `[optional]` 관리자/학생 관리 화면

### 2. [core] 학생 확인 모달

- 목적: 선택 학생과 캐릭터 확인.
- 버튼:
  - 시작하기
  - 다시 선택
  - 남자 캐릭터
  - 여자 캐릭터
- 이동:
  - 시작하기 -> 출근 경로 브리핑
  - 다시 선택 -> 학생 선택
- 전달 데이터: 선택 학생, 캐릭터 변경값.

### 3. [core] 출근 경로 브리핑

- 화면 ID: `screen-briefing`
- 목적: 오늘의 출근 목표, 버스, 경로, 이동 시간 확인.
- 버튼:
  - 출근 정보 입력하기
  - `다` 수준 단계별 확인/다시 듣기 버튼
- 이동:
  - 완료 -> AI 출근 계획 입력
- 전달 데이터: 목표 출근 시간, 추천 출발 시간, 버스 번호, 정류장, 하차 정류장, 이동 시간.

### 4. [core] AI 출근 계획 입력

- 화면 ID: `screen-ai-plan`
- 목적: AI에게 출근 정보를 입력하거나 확인시키고 계획을 계산.
- 입력:
  - 출근 시간
  - 출발 정류장
  - 도착 정류장
  - 버스 번호
  - 버스 도착 시간
  - 버스 이동 시간
- 버튼:
  - 출근 정보 다시 보기
  - AI가 계산해주기
  - 출근 정보 입력 완료(`가` 수준에서 계산 후 활성화)
- 이동:
  - 계산 완료 -> AI 계획 결과
- 전달 데이터: `aiCommutePlan`, 출발 시간, 기상 시간, 탑승 시간, 도착 예상 시간.

### 5. [core] AI 출근 계획 결과

- 화면 ID: `screen-ai-plan` 내부 결과 view
- 목적: AI 계산 결과를 타임라인으로 확인.
- 버튼:
  - 입력 다시 보기
  - 전날 준비 시작하기
- 이동:
  - 전날 준비 시작하기 -> 전날 준비 화면

### 6. [core] 전날 준비

- 화면 ID: `screen-eve-prep`
- 목적: 버스 정보 확인, 전날 가방 챙기기, 알람 설정.
- 내부 단계:
  - 버스 정보 확인
  - 가방 챙기기
  - 잠자리/알람 설정
- 버튼:
  - 버스 정보 확인 완료
  - 가방 완료
  - 알람 시간 선택 또는 10분 단위 조정
- 이동:
  - 알람 저장 -> 수면 전환 화면
- 전달 데이터: `evePrep.busChecked`, `evePrep.bagPacked`, `evePrep.alarmTime`.

### 7. [core] 수면 전환

- 화면 ID: `screen-sleep-transition`
- 목적: 밤에서 아침으로 시간이 흐르는 컷신 제공.
- 버튼: 없음. 타이머 기반 자동 이동.
- 이동:
  - 자동 -> 아침 알람 화면
- 전달 데이터: 알람 시간, 현재 시간.

### 8. [core] 아침 알람

- 화면 ID: `screen-morning-alarm`
- 목적: 알람 울림 후 바로 일어날지 10분 더 잘지 선택.
- 버튼:
  - 바로 일어나기
  - 10분 더 자기
- 이동:
  - 바로 일어나기 -> 아침 준비 화면
  - 10분 더 자기 -> 수면 전환 후 다시 아침 알람
- 전달 데이터: 실제 기상 시간, snoozeCount, wokeUpEarly.
- 상태 확정: 10분 더 자기 선택 시 `wokeUpEarly = false`.

### 9. [core] 아침 준비

- 화면 ID: `screen-morning-prep`
- 목적: 출근 전 필수 루틴 수행.
- 버튼:
  - 샤워하기
  - 아침 먹기
  - 양치하기
  - 옷 입기
  - 가방 챙기기
  - 오늘 출근 정보
  - 도움말
  - 출근 시작하기/출근 정보 확인하기
- 이동:
  - 각 활동 선택 -> 활동 컷신 -> 완료 확인 -> 아침 준비 메인
  - 가방 챙기기 -> 가방 미니게임 overlay
  - 모든 활동 완료 -> 통근 화면
- 전달 데이터: 완료 활동 목록, 현재 시간, dressedForWork, bagChecked.
- 상태 확정: 필수 준비물 완료 + 가방 완료 버튼 클릭 시 `bagChecked = true`.

### 10. [core] 가방 챙기기 미니게임

- 화면 ID: `screen-morning-prep` 내부 overlay
- 목적: 필수/상황 준비물을 가방에 넣고 방해물을 거르기.
- 버튼/동작:
  - 물건 클릭
  - 물건 드래그 앤 드롭
  - 가방 다 챙겼어요
- 이동:
  - 완료 -> 아침 준비 메인
- 전달 데이터: `homeBag`, `morningPrep.bagPackedItems`, 누락/오답 피드백 후보.

### 11. [core] 출근 이동

- 화면 ID: `screen-commute`
- 목적: 정류장 이동, 버스 선택, 하차, 목적지 찾기.
- 내부 단계:
  - 시작/정류장 방향
  - 정류장까지 걷기
  - 버스 번호 선택
  - 교통카드 확인 이벤트
  - 버스 탑승
  - 정류장 진행/하차벨
  - 목적지 선택
  - 회사까지 걷기
  - 도착
- 버튼:
  - 정류장 방향 선택
  - 정류장까지 걷기
  - 버스 타기
  - 기다리기
  - 집에 다녀오기
  - 다음 정류장으로 이동
  - 하차벨 누르기
  - 버스에서 내리기
  - 지도 목적지 hotspot
  - 회사까지 걷기
- 이동:
  - 도착 -> 결과 컷신
- 전달 데이터: commuteRecord, homeDepartureTime, stopArrivalTime, busBoardingTime, arrivalTime, missedBusTimes.

### 12. [core] 결과 컷신

- 화면 ID: `result-cutscene-overlay`
- 목적: 주요 피드백 사유에 맞는 상황 이미지와 메시지를 먼저 보여줌.
- 버튼:
  - 결과 확인
- 이동:
  - 결과 확인 -> 결과 화면

### 13. [core] 결과 화면

- 화면 ID: `screen-result`
- 목적: 최종 판정과 학습 피드백 표시.
- 버튼:
  - 결과 다시 듣기
  - 자세한 행동 기록 보기
  - 다시 하기
  - 교사용 리포트 보기
  - 처음으로
  - `[optional]` 오늘의 직무 하러 가기
- 이동:
  - 다시 하기 -> 같은 학생으로 게임 재시작
  - 처음으로 -> 학생 선택
  - 교사용 리포트 보기 -> 빠른 리포트 모달
  - 오늘의 직무 하러 가기 -> `[optional]` 직무 수행 화면
- 전달 데이터: rawJudgment, displayJudgment, feedback, actionLog, saved history.
- 결과 상태: 저장/분석은 `rawJudgment`, 화면 표시는 `displayJudgment`를 사용한다. `rawJudgment = incomplete`이면 `displayJudgment = success`로 표시 가능하다.

### 14. [core] 교사용 빠른 리포트 모달

- 화면 ID: `teacher-report-modal`
- 목적: 결과 화면에서 핵심 어려움과 다음 지도 포인트를 빠르게 확인.
- 버튼:
  - 닫기
  - 상세 보고서 보기
- 이동:
  - 상세 보고서 보기 -> 교사용 상세 리포트 화면

### 15. [core] 교사용 상세 리포트

- 화면 ID: `screen-report`
- 목적: 개별 학생 보고서와 전체 요약 확인 및 인쇄.
- 버튼:
  - 이전 화면으로
  - 개별 보고서 탭
  - 전체 요약 탭
  - 리포트 인쇄
  - 다시 하기
  - 학생 선택 화면으로 이동
- 이동:
  - 이전 -> 결과 화면
  - 다시 하기 -> 같은 학생으로 재시작
  - 홈 -> 학생 선택

### 16. [optional/post-result] 직무 수행

- 화면 ID: `screen-duty`
- 목적: 도착 후 오늘의 직무 2개를 체크리스트로 수행.
- 버튼:
  - 직무 카드 체크
  - 오늘의 직무 완료
  - 처음으로
- 이동:
  - 완료/처음으로 -> 학생 선택
- 전달 데이터: todayDuties checked 상태.

### 17. [optional/teacher-settings] 관리자/학생 관리

- 화면 ID: `screen-admin`
- 목적: 학생 추가/수정/삭제와 수준/성별/아바타 관리.
- 버튼:
  - 학생 추가
  - 뒤로
  - 학생별 수정/삭제/수준 변경 컨트롤
- 이동:
  - 뒤로 -> 학생 선택

## 화면 간 데이터 전달

- `student`: 학생 선택과 캐릭터 확인 후 전체 게임 상태에 저장.
- `todayDuties`: `[optional]` 게임 시작 시 2개 무작위 선택 가능. v2 MVP 핵심 판정에는 사용하지 않고 post-result 직무/리포트 확장 데이터로 사용한다.
- `todayInfo`: 출근 시간, 날씨, 준비물, `[optional]` 직무별 준비물, 주의사항.
- `busInfo`: 정류장, 버스 번호, 배차 시간, 하차 정류장, 목적지.
- `aiCommutePlan`: 기상/출발/탑승/하차/도착 예상 시간.
- `evePrep`: 알람 시간, 전날 챙긴 물건, 버스 확인 여부, 10분 더 자기 횟수.
- `morningPrep`: 완료 활동, 옷 상태, 가방 확인, 누락 물건, 피드백 후보.
- `commuteRecord`: 정류장/버스/하차/목적지 선택 기록과 오답 횟수.
- `feedback`: 결과 화면과 교사용 리포트의 핵심 어려움/지도 포인트.
- `rawJudgment`: 저장/분석용 원본 결과. 값은 success, incomplete, late, fail.
- `displayJudgment`: 화면 표시용 결과. `rawJudgment = incomplete`인 경우 success로 표시 가능.
- `actionLog`: 시간별 행동 기록.
- `history`: localStorage에 저장되는 완료 게임 결과.

## 자체 검증

- 시작부터 종료까지 현재 주요 흐름을 모두 포함했다.
- 결과 이후 교사용 리포트는 `[core]`로, 직무 수행은 `[optional/post-result]`로 분리했다.
- 관리자 화면은 `[optional/teacher-settings]`, 레거시 화면은 `[legacy/exclude]`로 분리했다.
- 각 화면별 버튼과 이동 조건을 빠짐없이 점검했다.
